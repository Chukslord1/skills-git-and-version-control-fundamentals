# Introduction to Git and Version Control

_Learn the core ideas behind version control by making real changes in a GitHub repository._

## Welcome

- **Who is this for**: Beginners with no prior Git or version control experience.
- **What you'll learn**: How to create commits, work in branches, open pull requests, merge changes, and safely revert mistakes.
- **What you'll build**: A team handbook in Markdown while practicing a complete Git workflow.
- **Prerequisites**:
  - A GitHub account
  - Basic familiarity with editing files on GitHub
- **How long**: This exercise takes less than 30 minutes to complete.

In this exercise, you will:

1. Make your first tracked change and commit it to the repository.
1. Create a feature branch and isolate work from the stable branch.
1. Open and merge a pull request, then inspect commit history.
1. Recover from a mistake by creating a revert commit.
